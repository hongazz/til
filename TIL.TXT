1. 앞으로 git 프로젝트로 만들고 싶은 것 
주제는 아직 정하지 못했으나 하나의 홈페이지를 여러사람들이랑 같이 만들어 보는것.

2. 1주차 주요 개념 키워드 적어보기
 
버전관리에 대한 내용 : 프로그램 상태가 변경되었을 경우 그것에 대한 정보를 알고있다는 것. 
버전관리 도구로 GIT이라는 프로그램이 있고, GITHUB를 통하여 협업을 할 수 있다.
GIT을 처음 사용할 시에 GIT INITIALIZE를 해야한다. (깃 초기화)
그리고 커밋을 하여 현재의 프로젝트의 상태를 저장.

커밋을 하기전에 스테이징으로 어떤 항목을 커밋할지 정할 수 있다.
커밋할 때, 커밋 메시지를 작업의 내용으로 적는것이 중요하다. 
GIT으로 관리하는 공간은 로컬 repo와 원격 repo로 나눌 수 있다.

로컬 repo에 있는 데이터를 원격 repo로 옮겨서 사용할 시에는 
트래킹을 하여, 로컬 repo와 원갹 repo를 동기화 한다.

원격 repo에 있는 데이터를 이용하고 싶을 경우에는 
로컬 repo에 데이터를 다운로드(클론) 한다.

로컬 repo에서 원격 repo로 데이터를 보낼 때는 push를 사용하고
원격 repo에서 로컬 repo로 데이터를 받을 때는 pull을 사용한다.
